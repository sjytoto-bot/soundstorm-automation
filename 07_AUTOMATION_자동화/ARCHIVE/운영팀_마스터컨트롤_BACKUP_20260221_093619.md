## 1️⃣ 역할 정의

마스터컨트롤은 SOUNDSTORM의 최상위 규칙 레이어를 담당한다. - 네이밍 / ID
체계 고정 - 자산 기준 확정 - OS 반영 기준 관리 - 에이전트 구조 설계 -
장기 브랜드 자산 방향 통제

이 팀은 실행팀이 아니라 "기준 고정 팀"이다.

### 마스터 자산 전환 기준

목적: 유튜브 제작용 음원 파일을 장기 라이브러리 자산으로 전환. 이
파일들은 SS_음원마스터 최종 시트 등록 대상.

고정 기준: 1. 파일명 표준 = 마스터 규칙과 동일 유지 2. ID = SS### 체계
고정 3. BPM 표기 방식 통일 4. 버전 표기 = 파일명 금지 → 메타데이터 관리
5. 공백 및 특수문자 규칙 고정

실행 금지: - 리네임 금지 - 자동화 수정 금지 - 메타데이터 구조 변경 금지

### \[음원 작업 구분 원칙 고정\] (v2.7 추가 블록)

SOUNDSTORM 기존 음원 작업은 다음 두 범주로 구분한다.

1.  편집(Edit)

-   기존 구조 유지 범위
-   자르기 / 붙이기 / 배열 변경 / 길이 조정
-   음악적 재설계 없음

2.  편곡(Arrangement)

-   구조 재설계
-   안무 최적화 포함
-   드롭/빌드업 재구성
-   악기 추가 가능
-   음악적 창작 개입 포함

안무 최적화 작업은 편곡에 해당한다.

가격 정책은 네이버스토어팀 레이어에서 관리한다. 마스터컨트롤은 구분
원칙만 고정한다.

### \[Strict Merge 의무화 규칙 고정\] (v2.9 추가 블록)

앞으로 다음 문서에 대해:

-   SOUNDSTORM_AI_OS
-   SOUNDSTORM_MASTER_ROADMAP
-   SOUNDSTORM_DATA_RULES
-   7개 팀가이드

추가 / 수정 / 삭제 / 블록 삽입이 발생할 경우 반드시 Strict Merge
Protocol로 실행한다.

필수 조건:

1.  원문 100% 유지
2.  Version +0.1 증가
3.  Timestamp 갱신
4.  Change Log 추가
5.  누락 검사 보고 포함
6.  파일 새로 생성 후 제공

파일 원문이 확보되지 않은 상태에서는 절대 수정하지 않는다. 자리표시자
보존 블록 생성 금지. 추정 병합 금지.

Strict Merge 수행을 위해 최신 파일 원문 확보를 기본 절차로 한다.
