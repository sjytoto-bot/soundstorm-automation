## 2. 통합 가능 여부 판단

### 2.1 통합 불가 탭 (구조 충돌)

| 탭 | 이유 |
|---|---|
| `_RawData_Master` | 영상 단위 원본 — 절대 건드리지 않음 |
| `_RawData_FullPeriod` | 차원 분석 append-only — 스키마 이질적 |
| `_Analytics_Snapshot` | Dashboard 전용 최적화 시트 — 독립 유지 |
| `Channel_KPI` | 시계열 추적 전용 — 다른 시트와 스키마 완전 다름 |
| `SS_음원마스터_최종` | 최종 마스터 소비층 — 보호 컬럼 구조 유지 필요 |
| `Thumbnail_Analysis` | CTR 동기화 타깃 — studio_csv_ingestor 직접 write |
| `Channel_CTR_KPI` | Video_Diagnostics의 기준선 소스 — 의존 관계 유지 |

### 2.2 통합 가능 탭

#### A. Analytics 4탭 → 1탭 (핵심 통합)

```
현재:
  Analytics_7d     — 11행 (SUMMARY 1 + VIDEO 10)
  Analytics_30d    — 11행 (SUMMARY 1 + VIDEO 10)
  Analytics_prev30 — 11행 (SUMMARY 1 + VIDEO 10)
  Analytics_all    — 11행 (SUMMARY 1 + VIDEO 10)
  합계: 44행 (시트 4개)

통합 후:
  Analytics_Periods — 44행 (period 컬럼 추가)
  합계: 44행 (시트 1개)
```

**추가 컬럼**: `period` ("7d" | "30d" | "prev30" | "all")

#### B. 수동 탭 2개 → 별도 스프레드시트 이동

```
이동 대상:
  업로드 예정
  트랜드분석&인사이트

이동 위치:
  새 스프레드시트: "SOUNDSTORM_운영시트"
  (자동화 시트와 완전 분리)
```

#### C. `_RawData_Recent30` 확인 후 처리

- 생성 주체 불명확 → 삭제 또는 유지 결정 필요

## 3. 통합 설계: Analytics_Periods

### 3.1 신규 시트 스키마

```
현재 Analytics_* 컬럼:
  type | key | views | likes | watch_time_min | avg_duration_sec |
  subscriber_change | ratio | rank | title

통합 후 Analytics_Periods 컬럼:
  period | type | key | views | likes | watch_time_min | avg_duration_sec |
  subscriber_change | ratio | rank | title
  ↑ period 컬럼 1개 추가
```

### 3.2 데이터 예시

| period | type | key | views | ... |
|---|---|---|---|---|
| 7d | SUMMARY | total | 12,450 | ... |
| 7d | VIDEO | ZgCeY-blMhs | 3,200 | ... |
| 30d | SUMMARY | total | 48,200 | ... |
| 30d | VIDEO | ZgCeY-blMhs | 11,800 | ... |
| prev30 | SUMMARY | total | 41,000 | ... |
| all | SUMMARY | total | 143,912 | ... |
| all | VIDEO | LbvbdVN8te8 | 28,400 | ... |

### 3.3 api_data_shuttler.py 수정 범위

```python
# 현재 코드 (4개 시트 개별 write)
aggr_dates = [
    ('Analytics_7d',     ...),
    ('Analytics_30d',    ...),
    ('Analytics_prev30', ...),
    ('Analytics_all',    ...),
]
for sheet_name, s_dt, e_dt in aggr_dates:
    ws_agg = ss.worksheet(sheet_name)
    ws_agg.clear()
    ws_agg.update(...)

# 변경 후 (단일 시트 write)
aggr_dates = [
    ('7d',     ...),
    ('30d',    ...),
    ('prev30', ...),
    ('all',    ...),
]
all_rows = []
for period_key, s_dt, e_dt in aggr_dates:
    # 각 period 행에 period 컬럼 추가
    rows = fetch_period_rows(period_key, s_dt, e_dt)
    all_rows.extend(rows)

ws_periods = ss.worksheet('Analytics_Periods')
ws_periods.clear()
ws_periods.update([AGGR_COLS] + all_rows)
```

**수정 라인**: `api_data_shuttler.py` L1261~L1334 (aggr_dates 루프 전체)

### 3.4 AnalyticsAdapter.ts 수정 범위

```typescript
// 현재 코드
const analyticsSheets =
  period === "7d"  ? ["Analytics_7d",  "Analytics_prev30", "Analytics_all"] :
  period === "30d" ? ["Analytics_30d", "Analytics_prev30", "Analytics_all"] :
                    ["Analytics_all"];

const [analyticsResult, ...] = await Promise.all([
  api.fetchSheetVideos(analyticsSheets),
  ...
]);

const allRows    = analyticsResult["Analytics_all"] ?? [];
const currentKey = period === "7d" ? "Analytics_7d" : "Analytics_30d" ...
const currentRows = analyticsResult[currentKey] ?? [];

// 변경 후
const [periodsResult, ...] = await Promise.all([
  api.fetchSheetVideos(["Analytics_Periods"]),
  ...
]);

const rawPeriods = periodsResult["Analytics_Periods"] ?? [];
// period 컬럼으로 필터링
const currentRows = rawPeriods.filter(r => r["period"] === period);
const prev30Rows  = rawPeriods.filter(r => r["period"] === "prev30");
const allRows     = rawPeriods.filter(r => r["period"] === "all");
```

**수정 파일**: `src/adapters/AnalyticsAdapter.ts` L758~L862

## 5. 구현 우선순위 및 작업 순서

### Phase 1 — 수동 탭 분리 (30분, 위험도 없음)

1. Google Sheets에서 `업로드 예정` 탭 내용 복사
2. 새 스프레드시트 "SOUNDSTORM_운영시트" 생성
3. 붙여넣기 후 원본 탭 삭제
4. `트랜드분석&인사이트` 동일 처리

### Phase 2 — `_RawData_Recent30` 확인 (10분)

1. 탭 내용 확인 → 컬럼 구조 분석
2. `_RawData_Master`와 중복 여부 판단
3. 미사용 확인 시 삭제

### Phase 3 — Analytics_Periods 통합 (2~3시간)

**Step 3-1**: `api_data_shuttler.py` 수정
- aggr_dates 루프에서 sheet_name → period_key 변경
- 단일 `Analytics_Periods` 시트로 write
- 기존 4탭 유지 상태로 병렬 write (롤백용)

**Step 3-2**: `AnalyticsAdapter.ts` 수정
- `Analytics_Periods` 단일 fetch 후 period 필터링

**Step 3-3**: 패널 테스트
- KpiCardsPanel (7d / 30d 전환)
- GrowthPanel (성장율 계산)
- HitVideosPanel (전체 기간 영상 순위)

**Step 3-4**: 구 탭 삭제
- Analytics_7d / 30d / prev30 / all 삭제
