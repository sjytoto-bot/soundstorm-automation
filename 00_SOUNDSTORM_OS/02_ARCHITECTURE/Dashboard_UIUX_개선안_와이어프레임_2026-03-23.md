## 1. 목적

이 문서는 현재 대시보드를 "정보판"에서 "실행 OS"로 더 강하게 보이도록 재구성하는 UI/UX 와이어프레임 문서다.

핵심 목표는 3가지다.

1. 진입 3초 안에 오늘 해야 할 1개 행동이 보이게 만들기
2. 상단 카드들의 역할 중복을 줄이고 정보 위계를 선명하게 만들기
3. 우측 패널과 좁은 화면에서도 메인 진단 영역이 무너지지 않게 하기

---

## 2. 핵심 UX 원칙

### Action First

- 첫 화면에서 가장 큰 시각 비중은 "지금 실행할 1개 행동"에 준다.
- 설명은 행동 뒤에 온다.

### Why Before Detail

- 사용자는 먼저 "왜 지금 이 행동이 중요한지"를 알아야 한다.
- 세부 수치와 드릴다운은 2차 정보다.

### One Screen Decision

- 스크롤 전 화면에서 아래 4개는 보여야 한다.
  - 긴급 경고 여부
  - 오늘 해야 할 1개 행동
  - 오늘 업로드/대기 판단
  - 채널 상태 요약

### Stable Layout

- 경고가 나타나도 전체 레이아웃이 크게 흔들리지 않아야 한다.
- 우측 패널 open/close가 메인 영역 가독성을 해치지 않아야 한다.

## 4. 섹션별 와이어프레임

### A. Critical Alert Banner

역할:
- 현재 시스템에서 가장 위험한 이슈 1개를 즉시 노출

와이어프레임:

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ [CRITICAL] CTR 급락: 최근 영상 1건                                           │
│ 썸네일/제목 수정이 필요합니다.                                                │
│                                                  [진단 열기]                 │
└──────────────────────────────────────────────────────────────────────────────┘
```

개선 원칙:
- CTA는 1개만 유지
- 본문은 1줄 설명으로 제한
- 배너가 없을 때 높이 점프가 최소화되도록 reserve space 고려

---

### B. Primary Action Hero

현재 대응:
- `ActionCommandBar`를 확장해 hero 영역으로 승격

목표:
- "오늘 해야 할 1개 행동"을 화면에서 가장 먼저 읽히게 함

와이어프레임:

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ TODAY ACTION                                                                 │
│ CTR 낮은 영상 1건 먼저 점검하세요                                             │
│ 최근 48시간 기준 클릭률이 채널 평균보다 낮습니다                              │
│ [CTR 진단 열기]                    보조: [업로드 판단] [채널 상태 보기]      │
└──────────────────────────────────────────────────────────────────────────────┘
```

개선 포인트:
- primary 제목 폰트 1단계 확대
- 설명 1줄 추가
- secondary는 chip이 아니라 summary action card로 재설계
- tag보다 결과 지향 문구 사용

문구 예시:
- "지금 실행 →" 대신 "CTR 진단 열기"
- "전략 확인 →" 대신 "업로드 판단 보기"

---

### C. Today Decision Row

현재 문제:
- `TodayBriefCard`와 `NextUploadCard`가 모두 타이밍을 설명해 역할이 겹친다

개선 방향:
- 오늘 판단과 다음 계획을 분리

구조:

```text
┌──────────────────────────────┬──────────────────────────────────────────────┐
│ 오늘 업로드 판단              │ 다음 업로드 계획                            │
│ 오늘은 대기 / 업로드 / 긴급   │ 다음 추천 슬롯, 리듬 기준, 대안 슬롯        │
│ 근거 2개                     │ 주간 계획용 정보                            │
│ [왜 이런 판단인지 보기]      │ [주간 계획 열기]                           │
└──────────────────────────────┴──────────────────────────────────────────────┘
```

카드 1: Today Decision
- 답해야 할 질문: "오늘 올릴까 말까?"
- 출력:
  - 상태: 업로드 / 대기 / 긴급 업로드
  - 근거 2개
  - 행동 CTA 1개

카드 2: Next Upload Plan
- 답해야 할 질문: "다음 최적 슬롯은 언제인가?"
- 출력:
  - 추천 날짜/시간
  - 리듬 vs 골든아워 비교
  - 대안 슬롯

---

### D. Channel Pulse

현재 문제:
- 한 줄에 KPI가 많아 스캔성이 떨어짐

개선 방향:
- 핵심 2개만 상시 노출
- 나머지는 펼치기 또는 hover/focus drilldown

와이어프레임:

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 채널 상태 A   30일 조회수 128,000   채널 CTR 4.8%    [진단 보기] [상세 열기] │
│ 펼침 시: 구독자 변화 / 시청 시간 / 최근 7일 흐름                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

상시 노출 추천:
- 건강도 등급
- 30일 조회수
- 채널 CTR

펼침 시 노출:
- 구독자 변화
- 시청 시간
- 주간 변화율

좁은 폭

### G. Right Panel

역할 재정의:
- 메인 작업 공간이 아니라 보조 탐색 패널

원칙:
- 기본은 compact
- 현재 문제 있는 섹션만 badge 강조
- 각 섹션 첫 화면은 3~5개 핵심 항목만 노출

와이어프레임:

```text
┌──────────────────────────┐
│ 전략  4                  │
│ 시청자 분석              │
│ 인기 영상 20             │
│ 기회 영상                │
│ 시청유지율 3             │
│ 외부 유입                │
└──────────────────────────┘
```

개선 포인트:
- 아코디언 제목 줄 높이/리듬 통일
- badge 스타일 통일
- 각 섹션 CTA는 1개만 유지

## 5. Topbar / Sidebar 개선안

### Topbar

현재:
- 제목 + 진행률 위주

목표:
- 전역 상태 바 역할 수행

구성:

```text
[모드] Dashboard   [마지막 동기화] 23:10   [긴급도] HIGH   [진행률] 68%
```

추가 권장 정보:
- 오늘 긴급도
- 마지막 시트/API 동기화
- 활성 경고 개수

### Sidebar

개선 포인트:
- active 상태 대비 강화
- collapsed 상태 툴팁 강화
- 제품 탭과 운영 탭의 그룹 구분 검토

## 6. 반응형 기준

### Breakpoints

- `>= 1440px`
  - Right Panel open 허용
  - execution 2열 유지

- `1280px ~ 1439px`
  - Right Panel 기본 collapsed 권장
  - 상단 hero 우선

- `< 1280px`
  - execution 1열
  - secondary actions 세로 적층
  - Channel Pulse 간소화

## 7. 상태 피드백 규칙

### 버튼 상태

- hover: border/surface 변화
- pressed: contrast 증가
- loading: spinner 또는 "처리 중"
- success: inline check + 짧은 완료 문구

### 이동 피드백

- 스크롤 이동 후 target block glow 2초
- 상단 안내 문구 1줄
  - 예: "CTR 이슈 영상 1건을 펼쳤습니다"

### 완료 피드백

- "추적 시작됨"
- "진단 열림"
- "업로드 판단 갱신됨"

## 10. 최종 판단

UI/UX 개선의 핵심은 "더 많은 정보"가 아니라 "더 빠른 판단"이다.

따라서 다음 원칙으로 구현해야 한다.

1. 상단에서 1개의 행동이 가장 먼저 보여야 한다.
2. 시간 관련 카드는 오늘 판단과 다음 계획으로 분리해야 한다.
3. 진단은 메인 작업 공간, 우측 패널은 보조 탐색 공간이어야 한다.
4. 좁은 폭에서도 메인 진단 영역이 최우선으로 보호되어야 한다.

## 11. 2026-03-23 구현 반영 체크

### 해결됨

- 상단 중복 시간 카드 제거

- KPI 상세를 카드 내부가 아니라 우측 inspector로 이동
- 메인 진단 → 우측 패널 → `VideoDetailModal` 종착 흐름 강화
- `ChannelPulseRow`의 KPI 중복 문제 축소

### 부분 해결

- Right Panel compact UX
- interaction feedback 통일
- 작은 화면에서의 완전한 폭 보호

### 구현 보류

- 추가적인 카드 장식/시각 polish
- 세부 motion 문법 고도화
