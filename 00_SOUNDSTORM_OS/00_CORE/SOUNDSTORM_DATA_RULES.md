# 📜 SOUNDSTORM 데이터 관리 헌법 (v11.4 Final Stability)

본 문서는 SOUNDSTORM 시스템의 데이터 무결성을 보장하고, 사용자의 메타데이터를 절대적으로 보호하며, 초고속 분석 환경을 유지하기 위한 **최상위 운영 원칙**을 정의합니다.

---

## 🏗️ 1. 데이터 거버넌스 원칙 (Governance)

### 🔑 절대 식별자 (Absolute Key)
- 모든 지표 매핑과 데이터 동기화의 기준은 영상 제목이 아닌 **`video_id` (영상ID)**로 고정합니다. 제목 변경 시에도 지표 이력은 보존됩니다.

### 🛡️ 수동 데이터 보호 (Manual Shield)
- **보호 대상**: `곡명`, `상품ID`, `음원파일`, `영상파일`, `장르`, `BPM`
- **보호 원칙**: 자동화 엔진은 위 컬럼의 데이터를 절대 덮어쓰거나 수정할 수 없습니다. 이는 사용자의 고유한 편집 권한 영역입니다.

### ⚡ 초고속 업데이트 방식
- **Cell-by-Cell Update**: 전체 영역 덮어쓰기(`setValues`)를 폐지하고, 변경된 지표만 개별 셀에 기록(`setValue`)합니다.
- **효과**: 사용자가 설정한 하이퍼링크(파일 링크), 리치텍스트 서식, 메모 등이 유실되지 않고 100% 영구 보존됩니다.

---

## 📁 2. 시트 및 컬럼 정의

### 📊 SS_음원마스터_최종 (Master Layer)
- **지표 업데이트**: 조회수, 좋아요, 댓글, 공유수, 구독자 등 API 데이터만 업데이트됩니다.
- **정밀도 최적화**:
    - `좋아요율`: 소수점 4자리 반올림
    - `일평균시청시간(분)`: 소수점 3자리 반올림
- **C1 타임스탬프** 포맷 (업데이트됨):
    ```
    데이터수집: {data_fetched_at (YouTube API 수집 시각)} | 동기화: {sync_ran_at KST (Python 실행 시각)} | 상태: SUCCESS / FAIL
    ```
    - `데이터수집`: `_RawData_Master`의 `data_fetched_at` 최신값 — API 데이터 신선도 표시
    - `동기화`: 파이썬 sync 실행 시각 — 매 실행마다 갱신
    - `상태`: 정상 완료 시 `SUCCESS`, 예외 발생 시 `FAIL | {에러 메시지 앞 80자}`

**자동 동기화 대상 컬럼 (Apps Script syncMasterV10 동일):**

| 컬럼명 | 소스 | 유형 | 비고 |
|--------|------|------|------|
| `유튜브_제목` | `youtube_title` (_RawData_Master) | API 수집 | |
| `썸네일URL` | `썸네일URL` (_RawData_Master) | API 수집 | |
| `러닝타임(초)` | `runtime_sec` (_RawData_Master) | API 수집 | |
| `게시일` | `upload_date` (_RawData_Master) | API 수집 | |
| `조회수` | `views` | API 수집 | |
| `좋아요` | `likes` | API 수집 | |
| `댓글` | `comments` | API 수집 | |
| `공유수` | `shares` | API 수집 | |
| `구독자유입` | `subscribers_gained` | API 수집 | |
| `총시청시간(분)` | `total_watch_time_min` | API 수집 | |
| `평균시청시간(초)` | `avg_watch_time_sec` | API 수집 | |
| `노출수` | `impressions` | API 수집 | |
| `CTR(%)` | `ctr` | API 수집 | |
| `유튜브URL` | `https://www.youtube.com/watch?v={video_id}` | **파생** | 자동 생성 |
| `러닝타임` | `formatDuration(runtime_sec)` | **파생** | 예: `2:09` / `1:02:33` |
| `업로드경과일수` | `max(1, 오늘 - 게시일)` | **파생** | |
| `시청유지밀도` | `avg_watch_time_sec / runtime_sec` | **파생** | |
| `일평균시청시간(분)` | `total_watch_time_min / 업로드경과일수` | **파생** | 소수 3자리 |
| `시간보정유지가치` | `시청유지밀도 × 일평균시청시간(분)` | **파생** | |
| `좋아요율` | `likes / views` | **파생** | 소수 4자리 |
| `시청유지등급` | 임계값 기반 | **파생** | 0.45↑ 높음 / 0.3↑ 중간 / 미만 낮음 |
| `자산등급` | 임계값 기반 | **파생** | 5↑ 높음 / 2↑ 중간 / 미만 낮음 |

> **보호**: `곡명`, `상품ID`, `음원파일`, `영상파일`은 위 목록에서 절대 제외됩니다.
> **수식 셀 보호**: `=` 로 시작하는 셀은 파이썬 코드가 덮어쓰지 않습니다 (formula_based_derived 자동 감지).

### 📥 _RawData_Master (Source Layer)
- Python 수집기가 100% 관리하는 로우 데이터 적재 공간입니다.
- **주요 컬럼**: `views`, `likes`, `comments`, `shares`, `subscribers_gained`, `total_watch_time_min`, `upload_date`, `data_fetched_at` 등.

## 📈 3. 핵심 분석 지표 (Indicator Core)

| 지표명 | 계산 및 처리 방식 | 의미 |
| :--- | :--- | :--- |
| **시청유지밀도** | `평균시청시간(초) / 러닝타임(초)` | 콘텐츠의 몰입도 (0.45 이상 '높음') |
| **일평균시청시간** | `총시청시간(분) / 업로드경과일수` | 일일 자산 생성 기여도 (소수 3자리) |
| **시간보정유지가치** | `시청유지밀도 × 일평균시청시간` | **[최종 KPI]** 종합적인 자산 가치 지수 |
| **시청유지등급** | 시청유지밀도 ≥ 0.45 → 높음 / ≥ 0.3 → 중간 / 미만 → 낮음 | 콘텐츠 몰입 수준 등급 |
| **자산등급** | 시간보정유지가치 ≥ 5 → 높음 / ≥ 2 → 중간 / 미만 → 낮음 | 투자 및 마케팅 판단의 근거 |
| **업로드경과일수** | `max(1, 오늘 - 게시일)` | 최솟값 1일 보장 (당일 업로드 분모 0 방지) |
| **러닝타임** | `formatDuration(runtime_sec)` | 예: `2:09` / `1:02:33` (Apps Script 동일 형식) |

## ⚖️ 4. 시스템 금지 및 준수 사항 (Anti-Broken Rules)

1.  **시트 구조 파괴 금지**: 마스터 시트의 헤더명(2행)을 변경하거나 열을 임의로 삽입/삭제할 수 없습니다. (REQUIRED 검증 통과 필수)
2.  **서식 충돌 방지**: 스크립트는 시트의 서식(`setNumberFormat`)을 건드리지 않습니다. 모든 시각적인 스타일(드롭다운, 테두리, 배경색)은 사용자가 시트 메뉴에서 직접 관리합니다.
3.  **상품ID 수동 관리**: 시스템은 상품ID를 강제로 비우거나 생성하지 않습니다. 번호 체계는 사용자의 관리 규칙을 100% 따릅니다.
4.  **타임아웃 방어**: 드라이브 스캔 등 무거운 작업은 동기화 프로세스에서 배제하여 3초 이내 실행 속도를 유지합니다.
5.  **APPEND+UPDATE 분리 원칙**: SS_음원마스터_최종 동기화는 두 단계로 실행됩니다.
    - **UPDATE**: 이미 존재하는 video_id 행 → `batch_update` (셀 단위, 보호 컬럼 제외)
    - **APPEND**: `_RawData_Master`에만 있는 신규 video_id → `append_row` (보호 컬럼 빈 값 유지)
    - APPEND 완료 후 게시일(`upload_date`) 기준 내림차순 정렬 자동 실행
6.  **수식 셀 보호**: 파생 컬럼 셀이 `=`로 시작하는 경우 시트 내 수식으로 관리되는 것으로 간주합니다. 해당 컬럼은 파이썬 코드가 덮어쓰지 않습니다 (`formula_based_derived` 자동 감지).
